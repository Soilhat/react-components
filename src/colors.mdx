## Developer Usage Guide

When building or contributing a new component to `@soilhat/react-components`, **never write raw primitives or explicit `dark:` sub-variants** unless handling highly specialized custom visualizations.

### ❌ What NOT to do

```tsx
// This creates technical debt and prevents automated variable swapping across environments
<div className="bg-white dark:bg-[#161b22] border-gray-300 dark:border-gray-700">
  <p className="text-[#0d1117] dark:text-[#f0f2f5]">Content Block</p>
</div>
```

🎯 The Correct Approach

```TypeScript
// The semantic engine handles the dark mode variation transition seamlessly under the hood!
<div className="bg-card border border-border p-4 rounded-xl">
  <p className="text-foreground text-sm">Content Block</p>
  <p className="text-muted-foreground text-xs">Secondary metadata context indicator</p>
</div>
```
